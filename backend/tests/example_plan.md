# REST API Project — Full Workflow

---

## Workflow Steps

### [ ] Step: Requirements
Create a Product Requirements Document (PRD) based on the feature description.

1. Analyze the task description to understand what needs to be built
2. Identify requirements, user stories, and acceptance criteria
3. Save the PRD to `requirements.md`

### [ ] Step: Design API
  - [ ] Define REST endpoints and URL structure
  - [ ] Design database schema and relationships
  - [ ] Write OpenAPI 3.0 specification

### [ ] Step: Implementation
  - [ ] Set up project scaffold (Flask + SQLAlchemy)
  - [ ] Implement data models and migrations
  - [ ] Build CRUD API endpoints
  - [ ] Add JWT authentication middleware
  - Refer to spec.md for endpoint contracts (context note, not a task)

### [ ] Step: Testing
  - [ ] Write unit tests for models
  - [ ] Write integration tests for API endpoints
  - [-] Performance/load testing (skipped for MVP)

### [!] Step: Deployment (blocked on infra provisioning)

### [ ] Phase 2: Documentation
Write API docs and usage guide.
